Answers to the Basics

Question 1:

How important is logging in an application?

Answer:

Logging in any application is very important, you can log a lot of critical things and gain unprecedented insight into how your application is working and performing.


Question 2:

When do you think use of global variables is appropriate?

Answer:

When the scope of the variable is really small, and only use if it is really needed.


Question 3:

Name 3 DO’s and 3 DON’Ts around Exception Handling.

Answer:

DO's DON'Ts
Do use try/finally blocks around code that can potentially generate an exception and centralize your catch statements in one location. Don’t throw an exception when a simple if-else statement can be used to check for errors.
Do use the predefined exceptions types for common cases and define your own custom exception types only for programmatic scenarios. Don’t throw Exception(). It is too broad class and catching it could mean swallowing other exceptions. Instead create your own custom exceptions and derive from Exception class.
Do clean up intermediate results and resources when throwing an exception. Callers should be able to assume that there are no side effects when an exception is thrown from a method. Don’t swallow an exception by putting an empty catch block.
